一些其他的问题

  • 内部类不能被重写。当继承父类的子类拥有和父类同名的内部类时，这两个内部类将是完全不相干的两个内部类。
  • 本地内部类可以拥有构造器，也可以访问本地的final变量，而匿名内部类不可以拥有构造器，只允许实例初始化。

Interface:接口

发表于 2017-11-03 | 分类于 java学习笔记

接口

Interfaces and abstract classes provide more structured way to
separate interface from implementation.

接口和抽象类提供了一种分开接口和实现的更加结构化的方式。

抽象类和抽象方法

对于继承树上的基类，往往我们创建它们只是为了创建一个基本结构，而它们的接口方法一般都是没有实现的，即是一个空方法。为了规范，我们可以把这种方法写成抽象方法（abstract关键字），把这种类称作抽象类。

  • 如果一个类中含有抽象方法，则该类必须被声明为抽象类。
  • 抽象方法不能被调用，抽象类无法被创建对象。
  • 如果一个抽象类的子类没有重写所有的抽象方法，则该子类也应该被声明为抽象类。

Polymorphism:多形性

发表于 2017-10-22 | 分类于 java学习笔记

多形性

Polymorphism is the third essential feature of an object-oriented programming language, after data abstraction and inheritance.

多形性是除数据抽象以及继承以外第三关键的面向对象编程的特性。

利用具有多形性的方法调用，一种类型可将自己与另一种相似的类型区分开，只要它们都是从相同的基础类型中衍生出来的。这种区分是通过各种方法在行为上的差异实现的，可通过基础类实现对那些方法的调用。

向上造型

上一章已经提过，向上造型可以使我们传递参数时不仅可以传递基类的对象，还可以传递所有从基类继承的对象，因为它们共享着基类的全部接口方法。通过向上造型，我们可以不用为同一个类型下的不同的具体类分别写不同的调用函数，而是可以直接调用它们的基类，而这也是多形性的主要作用。

类的重用

“初始化”和“清理”是这些安全问题的其中两个。许多 C 程序的错误都是由于程序员忘记初始化一个变量造成的。对于现成的库，若用户不知道如何初始化库的一个组件，就往往会出现这一类的错误。清理是另一个特殊的问题，因为用完一个元素后，由于不再关心，所以很容易把它忘记。这样一来，那个元素占用的资源会一直保留下去，极易产生资源（主要是内存）用尽的后果。Java 也使用构造器和垃圾收集器来解决这些问题。

Everything is an Object:万物皆为对象

发表于 2017-09-26 | 分类于 java学习笔记

Java与面向对象

Although it is based on C++, Java is more of a “pure” object-oriented
language.

尽管基于C++，java相较之下更能称之为一种纯粹的面向对象的语言。

由于C++需要对C语言提供向下兼容性，这导致C++包含了许多其自身不期望的特性，这使得C++在某些方面变得过于复杂。然而Java语言确保你所做的完全是面向对象的编程，因此在开始着手于Java的学习之前应该首先将思维完全切换到面向对象的世界中去。

引用(Reference)

You treat everything as an object, using a single consistent
syntax. Although you treat everything as an object, the identifier you manipulate is actually a
“reference” to an object.

在Java中，所有的东西都应该被看作一个对象，你应该用相同的语法调用他们。也正因为你把这些东西都看作对象，你操作的实际上是这个对象的一个引用。

You might imagine a television (the object) and a remote control
(the reference). As long as you’re holding this reference, you have a connection to the
television, but when someone says, “Change the channel” or “Lower the volume,” what you’re
manipulating is the reference, which in turn modifies the object. If you want to move around the room and still control the television, you take the remote/reference with you, not the
television.

举个例子，你可以想象一个电视机(对象)和一个遥控器(引用)。你操作引用，如换台和调音量，最终导致对象的改变。每当你想换个地方坐着看电视的时候，你只要拿着遥控器(引用)而不用直接拿着电视机(对象)。

Operators And Controlling Execution:运算符与流程控制

发表于 2017-09-30 | 分类于 java学习笔记

由于Java继承自C，因此Java的运算符与控制结构都与C有异曲同工之处。下面就分两部分简单介绍Java的运算符和流程控制。

运算符

运算符以一个或多个自变量为基础，可生成一个新值。除此以外，一个运算符可改变运算对象的值。几乎所有运算符都只能操作基本类型。唯一的例外是”=”，”==”和”!=”，它们能操作所有对象。除此以外， String 类支持”+”和”+=”。

赋值

赋值是用等号运算符(=)进行的。它的意思是“取得右边的值，把它复制到左边”。右边的值可以是任何常数、变量或者表达式，但左边的值必须是一个明确的、已命名的变量。例如可将一个常数赋给一个变量(A=4)，但不可将任何东西赋给一个常数(不能4=A)。

基本类型的赋值单纯地将值从一个地方复制到另一个地方，而对对象的赋值则没有这么简单。由于对象创建的实例传递时使用的是引用，因此例如将D赋值到C(C=D)会将D所持有的引用复制到C，换言之C，D都指向了原来只有D指向的对象。例如：

算术运算符

Java的基本算术运算符与其他大多数程序设计语言是相同的。其中包括加号（+）、减号（-）、除号（/）、乘号（*）以及取模（%）。Java 也用一种简写形式进行运算，并同时进行赋值操作。这是由等号前的一个运算符标记的，而且对于语言中的所有运算符都是固定的。例如，为了4加到变量x，并将结果赋给x，可用：x+=4。

自动递增与递减

与C相同的是，Java提供简便的运算符。最经典的莫过于自动递增与递减运算符。”++”与”–”分别代表“增加一个单位”和“减少一个单位”。另外，当他们作为变量的前缀，例如”++x”时，代表先执行运算，再生成值，即先加后赋值；作为后缀的时候则相反，先赋值，再执行运算。

关系运算符

关系运算符常运用于比较两个运算对象。关系运算符包括小于（<）、大于（>）、小于等于（<=）、大于或等于（>=）、等于（==）以及不等于（!=）。等于和不等于适用于所有内建的数据类型，但其他比较不适用于 boolean 类型。

按位运算符

按位运算符允许我们操作一个整数的基本类型中的单个二进制位。按位运算符会对两个自变量中对应的位执行布尔代数，并最终生成一个结果。

  • 按位与（&），按位或（|），按位非（~）符合逻辑运算的真值表。
  • 按位异或(^)当两个运算对象相等在该位生成0，否则生成1。
  • 按位运算符可与等号（=）联合使用，以便合并运算及赋值：&=，|=和^=都是合法的（由于~是一元运算符，所以不可与=联合使用）。
  • 布尔型（boolean）由于只有一位，因此比较特殊。为了不和逻辑非相混淆，不能对布尔型执行按位非，但是其他的按位运算符均可执行，而且不会有逻辑短路的情况发生。另外，在下面一节使用移位运算符的时候不能使用布尔型。

移位运算符

移位运算符面向的运算对象也是二进制的位。可用它们来处理整数类型。

  • 左移位运算符（<<）将运算符左边的运算对象向左移动运算符右侧指定的位数（在低位补 0）。
  • 有符号右移位运算符（>>）将运算符左边的运算对象向右移动运算符右侧指定的位数。“有符号”右移位运算符使用了“符号扩展”：若值为正，则在高位插入0；若值为负，则在高位插入1。
  • 无符号右移位运算符（>>>）将运算符左边的运算对象向右移动运算符右侧指定的位数。它使用了“零扩展”：无论正负，都在高位插入0。

若对char，byte或者short进行移位处理，那么在移位进行之前，它们会自动转换成一个int。对于int，移位数不能大于32位（否则全移动完就没有意义了），因此操作符右侧数只有低5位有效。若对一个long进行处理，最后得到的结果也是long，同理此时只会用到操作符右侧数的6个低位（移动64位）。

移位可与等号（<<=或>>=或>>>=）组合使用。此时，运算符左边的值会移动由右边的值指定的位数，再将得到的结果赋回左边的值。

当移位运算符与等号组合使用时，移动char，byte，short会出现问题。它们被转化为int移位完后却在转回原来类型的时候损失了信息，因此这样使用会使移位结果变为-1。

在 C 和 C++中，强制转换有时会让人头痛。在 Java 里，强制转换则是一种比较安全的操作。但是，若进行一种名为“缩小转换”的操作（也就是说，脚本是能容纳更多信息的数据类型，将其转换成容量较小的类型），此时就可能面临信息丢失的危险。此时，编译器会强迫我们进行强制转换。而对于“放大转换”，则不必进行强制转换，因为新类型肯定能容纳原来类型的信息，不会造成任何信息的丢失。

Java 允许我们将任何主类型“强制转换为其他任何一种主类型，但布尔值（bollean）要除外，后者根本不允许进行任何强制转换。“类”不允许进行强制转换。为了将一种类转换成另一种，必须采用特殊的方法，在之后的章节中我们会提到。

Java没有sizeof()

在 C 和 C++中， sizeof()运算符能满足我们的一项特殊需要： 获知为数据项目分配的字节数量。在C和C++中，sizeof()最常见的一种应用就是“移植”。不同的数据在不同的机器上可能有不同的大小，所以在进行一些对大小敏感的运算时，程序员必须对那些类型有多大做到心中有数。例如，一台计算机可用32位来保存整数，而另一台只用16位保存。显然，在第一台机器中，程序可保存更大的值。移植是令C和 C++程序员颇为头痛的一个问题。

Java 不需要 sizeof() 运算符来满足这方面的需要，因为所有数据类型在所有机器的大小都是相同的。我们不必考虑移植问题——Java本身就是一种“与平台无关”的语言。

循环语句

while， do-while 和 for 控制着循环，有时将其划分为“反复语句”。除非用于控制反复的布尔表达式得到“假”的结果，否则语句会重复执行下去。

while循环

在循环刚开始时，会计算一次“布尔表达式”的值。而对于后来每一次额外的循环，都会在开始前重新计算一次。

do-while循环

while 和 do-while 唯一的区别就是 do-while 肯定会至少执行一次，即便表达式第一次便计算为 false。而在 while 循环结构中，若条件第一次就为 false，那么其中的语句根本不会执行。在实际应用中， while 比 do-while 更常用一些。

for循环

for 循环在第一次反复之前要进行初始化。随后，它会进行条件测试，而且在每一次反复的时候，进行某种形式的“步进”（Stepping）。

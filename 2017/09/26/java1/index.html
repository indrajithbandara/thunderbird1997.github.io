Everything is an Object:万物皆为对象

发表于 2017-09-26 | 分类于 java学习笔记

Java与面向对象

Although it is based on C++, Java is more of a “pure” object-oriented
language.

尽管基于C++，java相较之下更能称之为一种纯粹的面向对象的语言。

由于C++需要对C语言提供向下兼容性，这导致C++包含了许多其自身不期望的特性，这使得C++在某些方面变得过于复杂。然而Java语言确保你所做的完全是面向对象的编程，因此在开始着手于Java的学习之前应该首先将思维完全切换到面向对象的世界中去。

引用(Reference)

You treat everything as an object, using a single consistent
syntax. Although you treat everything as an object, the identifier you manipulate is actually a
“reference” to an object.

在Java中，所有的东西都应该被看作一个对象，你应该用相同的语法调用他们。也正因为你把这些东西都看作对象，你操作的实际上是这个对象的一个引用。

You might imagine a television (the object) and a remote control
(the reference). As long as you’re holding this reference, you have a connection to the
television, but when someone says, “Change the channel” or “Lower the volume,” what you’re
manipulating is the reference, which in turn modifies the object. If you want to move around the room and still control the television, you take the remote/reference with you, not the
television.

举个例子，你可以想象一个电视机(对象)和一个遥控器(引用)。你操作引用，如换台和调音量，最终导致对象的改变。每当你想换个地方坐着看电视的时候，你只要拿着遥控器(引用)而不用直接拿着电视机(对象)。

基本类型(primative types)

One group of types, which you’ll use quite often in your programming, gets special treatment.
You can think of these as “primitive” types. The reason for the special treatment is that to
create an object with new—especially a small, simple variable—isn’t very efficient, because
new places objects on the heap. For these types Java falls back on the approach taken by C
and C++. That is, instead of creating the variable by using new, an “automatic” variable is
created that is not a reference. The variable holds the value directly, and it’s placed on the
stack, so it’s much more efficient.

因为使用new创建新对象的效率并不很高，因此Java将那些常用的类型定义为了基本类型。这些基本类型的变量直接保存值的本身而不是引用，并且被直接保存在栈中，这使得他们的效率变得更高。

高精度数

Java拥有两个可用于高精度计算的类：BigInteger和BigDecimal。这两个类类似包装类，不同的是它们并没有对应的基本类型。

  • BigInteger supports arbitrary-precision integers. This means that you can accurately
    represent integral values of any size without losing any information during operations.

    BigInteger类用来保存那些不希望在计算操作中损失精度的整数。

  • BigDecimal is for arbitrary-precision fixed-point numbers; you can use these for accurate
    monetary calculations, for example.

    BigDecimal类用来储存那些高精度定点数，可以用来准确计算如金融数据等。

Java 中的数组

为了确保安全，java在初始化数组时会确保所有的元素都被初始化，并且无法越界访问。虽然损失了一些性能，但是得大于失。

当创建了一个Java数组，你实际创建的是一系列的引用，且所有的引用都会被赋予一个特殊的值：null。Java语言默认null意味着引用未绑定对象。当你想要使用这个数组时，你必须要为这些元素赋值，否则Java会报告运行时错误。这些规定使Java避开了那些在使用数组上可能出现的错误。

对象的作用域和Java垃圾回收机制

Java对象的作用域不同于基本类型。一旦使用new创建一个新的对象，即使离开了作用域它也仍然存在。这使我们不需要在Java中主动地销毁一个不用的对象。

与此同时，为了防止新对象不断的增加从而导致系统内存耗尽，停止响应，Java自带了一个垃圾回收机制，它会搜索并自动释放那些不再被需要的对象的内存来为新对象腾出空间。

When you define a class, you can put two types of elements in your class:

定义一个类的时候，里面可以有两种元素：域（也称为数据成员）和方法（也称为成员函数）。

名称可见性

Java为了使程序的命名不发生冲突，引入了包的概念。包是由一些类组成的库。Java开发者希望将开发者将域名反过来写作为包名的前缀。例如一个域名为example.com，则包名为com.example。这样，因为域名是唯一的，所以包名也是唯一的，将相同的名称区别到不同的包中，解决了命名冲突的问题。

static 关键字

有时候，我们的程序会有以下需求：

  • 类中需要一个域，所有类创建的对象实例中该域共享一片存储空间（即他们的值始终相同）
  • 类中需要一个方法，我们希望这个方法无论该类是否创建了对象实例，都可以被调用。

这个时候，我们就要使用static关键字来定义这样的域和方法。

When you say something is
static, it means that particular field or method is not tied to any particular object instance of
that class. So even if you’ve never created an object of that class you can call a static method
or access a static field.

static(静态)意味着该域或方法与类的实例无关，因此你可以在没有创建类的实例的情况下调用静态方法和静态域。

发表于 2017-11-03 | 分类于 java学习笔记

接口

Interfaces and abstract classes provide more structured way to
separate interface from implementation.

接口和抽象类提供了一种分开接口和实现的更加结构化的方式。

抽象类和抽象方法

对于继承树上的基类，往往我们创建它们只是为了创建一个基本结构，而它们的接口方法一般都是没有实现的，即是一个空方法。为了规范，我们可以把这种方法写成抽象方法（abstract关键字），把这种类称作抽象类。

  • 如果一个类中含有抽象方法，则该类必须被声明为抽象类。
  • 抽象方法不能被调用，抽象类无法被创建对象。
  • 如果一个抽象类的子类没有重写所有的抽象方法，则该子类也应该被声明为抽象类。

接口

如果说抽象类只是将基类的一部分公共方法进行了抽象，而保留了其他的方法，那么接口则进一步将抽象发挥到了极致：接口只包含抽象方法和常数。也就是说，接口提供的是一种调用形式，而不是任何实现。

Java中的“多重继承”

Java中，我们最多可以继承一个类（抽象类或普通的类），但我们却可以在一个类中实现许多接口（数量没有限制）。

Hero类同时结合了三个接口CanFight,CanSwim,CanFly，并且可以同时向上造型为三个接口类型。这也是多重继承的主要目的。当我们在创建基类的时候，我们应该试着考虑是否可以将这个基类定义为接口。

同时，我们可以使用继承来拓展接口。另外，在接口继承接口的时候，extends关键字允许多重继承。

另外，我们应该尽量避免在会被同一个类使用的接口和抽象类/实体类中设计方法名称相同，只有返回类和参数类型不同的方法，这样会导致关系的错乱而得到意向不到的结果。

基于接口的这些特性，使得我们在使用接口时及其容易写出可随时拓展的代码。因为，无论如何，只要一个类实现了接口，就可以作为参数被传递并被向上造型为接口方法。这种便利对库的使用者十分有用，这使我们可以为库增添更多功能，或基于库的原有功能集成开发新的库。
